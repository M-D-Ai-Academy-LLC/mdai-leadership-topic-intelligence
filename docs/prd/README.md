# PRD Directory

This directory stores Product Requirements Documents (PRDs) for this repository.

## Status

`docs/prd/` existed but had no tracked files. This README and template were added to initialize it.

## Naming Convention

Use zero-padded numbering for easy sorting:

- `0001-prd-<feature-name>.md`
- `0002-prd-<feature-name>.md`

Example:

- `0001-prd-topic-intelligence-api-hardening.md`

## Minimum PRD Sections

1. Introduction and problem statement
2. Goals
3. User stories
4. Functional requirements
5. Non-goals
6. Technical considerations
7. Success metrics
8. Open questions

## Related Directory

- Task lists generated from PRDs should be saved in `docs/tasks/`.
